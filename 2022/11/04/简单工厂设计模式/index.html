简单工厂设计模式

简单工厂设计模式

简单工厂设计模式又称静态工厂方法（static factory method），它不属于23种设计模式之一。

简单工厂设计模式是由工厂确定创建出哪一种产品类的实例，是工厂模式家族中最简单的模式。

简单工厂设计模式的优缺点

  • 优点：

    1. 解决了客户端直接依赖于具体对象的问题，消除了客户端创建对象的责任，仅仅承担使用的责任，实现了对责任的分割。
    2. 简单工厂也起到了代码复用的作用

  • 缺点：

    1. 系统扩展困难，一旦加入新功能，就必须要修改工厂的逻辑
    2. 简单工厂集合了所有创建对象的逻辑，一旦不能正常工作，会导致整个系统出问题

案例演示

public class SimpleFactory {
    public static void main(String[] args) {
        System.out.println("请输入两个操作数：");
        Scanner scanner=new Scanner(System.in);
        double d1=scanner.nextDouble();
        double d2=scanner.nextDouble();
        System.out.println("请输入操作符：");
        String oper=scanner.next();
        ICalculator calculator=CalFactory.getICalculator(oper);
        double result = calculator.getResult(d1, d2);
        System.out.println(result);

    }
}

//静态工厂方法：将创建对象的过程封装到静态方法中，客户端直接调用，实现了客户端和创建对象的解耦
class CalFactory{
    public static ICalculator getICalculator(String oper){
        ICalculator calculator=null;
        switch (oper){
            case "+":
                calculator=new Add();
                break;
            case "-":
                calculator=new Sub();
                break;
            case "*":
                calculator=new Mul();
                break;
            case "/":
                calculator=new Div();
                break;

        }
        return calculator;
    }
}

//计算类的接口
interface ICalculator{
    //返回最终计算的结果
    public double getResult(double d1,double d2);
}
//加
class Add implements ICalculator{

    @Override
    public double getResult(double d1, double d2) {
        return d1+d2;
    }
}
//减
class Sub implements ICalculator{

    @Override
    public double getResult(double d1, double d2) {
        return d1-d2;
    }
}
//乘
class Mul implements ICalculator{

    @Override
    public double getResult(double d1, double d2) {
        return d1*d2;
    }
}
//除
class Div implements ICalculator{

    @Override
    public double getResult(double d1, double d2) {
        return d1/d2;
    }
}
